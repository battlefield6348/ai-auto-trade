# AI Auto Trade 系統 UI 重新設計需求說明書 (Redesign Master Spec)

本文件彙整了系統目前所有的核心功能與業務邏輯，供 Stitch 進行全方位的 UI/UX 重新設計。系統定位為一個「基於 AI 評分的自動加密貨幣交易平台」，設計風格應維持：**深色科技感 (Cyberpunk/Dark Mode)**、**高性能感 (High Performance)** 以及 **數據可視化 (Data Driven)**。

---

## 1. 全域導覽與架構 (Global Navigation)

### 10.1 側邊欄應包含以下模組：
1.  **數據面板 (Dashboard)**：系統運行概況與即時分析。
2.  **AI 優化器 (AI Optimizer)**：【新功能】自動參數追蹤與優化。
3.  **策略管理 (Strategy Management)**：策略庫的 CRUD 與狀態控制。
4.  **回測控制台 (Backtest Terminal)**：手動調優與歷史模擬。
5.  **自動交易監控 (Live Trading)**：實體/模擬倉位監控與交易紀錄。
6.  **系統設定 (System Config)**：Binance API 金鑰管理、環境切換。

### 1.2 全域工具欄 (Global Toolbar)：
- **環境切換器**：支援 `Test (測試網)`、`Paper (模擬盤)`、`Live (正式盤)`，需有明顯的顏色區分（如 Live 為琥珀色警告色）。
- **系統時鐘**：顯示伺服器時間。

## 2. 模組細節需求

### A. 數據面板 (Dashboard)
- **系統管線狀態**：顯示下一次自動分析的倒數計時。
- **指標卡片**：顯示最近一次分析的成功數、失敗數、執行耗時。
- **市場快照**：以 `BTCUSDT` 為主，顯示當前的 AI `BASE_SCORE` (0-100)、價格、24h 漲跌幅、以及成交量比。
- **操作日誌**：顯示系統背景工作 (Cron Jobs) 的近期紀錄。

### B. AI 優化器 (AI Optimizer) - 【核心新增項目】
- **參數掃描設定**：
  - 選擇幣種、回測天數 (60/90/180)。
  - **自動部署開關**：開啟後，系統會自動將回測績效第 1 名的策略「儲存並啟用」。
- **優化進度**：由於運算量大，需有酷炫的運算動畫。
- **推薦方案卡片**：展示優化後的最佳參數（門檻、停利停損、條件權重）及其預期總收益、勝率、交易筆數。

### C. 策略管理 (Strategy Management)
- **策略列表**：
  - 顯示名稱、Slug、進/出場門檻、更新時間。
  - **狀態切換**：Toggle 開關控制「啟用/草稿」。啟用後背景 Worker 會自動執行。
- **詳情檢視**：點擊可查看該策略包含的所有加權條件 (Scoring Rules)。

### D. 回測控制台 (Backtest Terminal)
- **手動調優**：左側面板可自訂進場與出場的權重條件 (MA 乖離、成交量比、漲跌幅等)。
- **績效圖表**：
  - **主圖表**：價格曲線與 AI 評分曲線的重疊圖。
  - **獲利分布**：統計命中後的 3/5/10 天平均回報率（長條圖）。
- **交易明細表**：列出每一筆模擬買賣的進場點、出場原因 (觸發門檻/止盈/止損) 與損益。

### E. 自動交易監控 (Live Trading)
- **資產總覽**：顯示目前連線環境的 USDT 餘額與可用資產。
- **持倉面板**：顯示目前持有的幣種、進場均價、當前獲利 (Unrealized PnL)。
- **交易歷史**：過往真實/模擬交易的成交紀錄。
- **緊急操作**：一鍵全平倉按鈕。

## 3. 業務邏輯提醒 (Business Logic)
1.  **雙軌評分制**：進場與出場是分開評分的。進場看 `Entry Score` (總分 >= 門檻則買)，出場看 `Exit Score` (總分 < 門檻則賣)。
2.  **交易成本**：所有回測與交易應考量 **0.1%** 的預設手續費。
3.  **停利停損**：所有策略皆有內建的硬性比例（如 TP 10%, SL -3%），需在介面中可供設定。
4.  **歸一化顯示**：所有計分項匯總後，應歸一化為 0-100 分，讓使用者直覺理解「多空強度」。

## 4. 交付物品要求 (Stitch Instructions)
1.  **全套 React/Tailwind 元件**。
2.  **高度一致性的佈局 (Layout)**：統一的側邊欄與頁首。
3.  **互動回饋**：成功操作、金鑰錯誤、系統 Loading 的視覺化呈現。
4.  **響應式設計**：雖然以 Desktop 為主，但需確保側邊欄在行動裝置可收摺。
